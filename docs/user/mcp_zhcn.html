Axiomate user guide

MCP 服务器配置与使用

更新时间：2026-06-02。本文说明如何在 Axiomate 里连接外部 MCP server、管理配置、处理认证和排查常见问题。

一句话：MCP server 是 Axiomate 之外的工具提供者。你把一个 stdio / HTTP / SSE server 加进配置后，Axiomate 启动时会连接它，读取它暴露的 tools、prompts 和 resources，并把工具以 mcp__server__tool 的形式加入可用工具池。

最常用入口是 axiomate mcp add 添加配置，进入对话后用 /mcp 查看状态、认证、启停和重连。

快速开始

Axiomate 支持三种面向普通用户的 MCP transport：stdio、http、sse。不传 --transport 时默认按 stdio 处理。

添加本地 stdio server

axiomate mcp add my-server -- npx my-mcp-server

stdio server 会由 Axiomate 拉起为子进程，通过标准输入输出和 Axiomate 通信。命令后面的 -- 用来告诉 CLI：后面的内容是 server 命令和参数，不是 Axiomate 自己的参数。

添加 HTTP server

axiomate mcp add --transport http sentry https://mcp.sentry.dev/mcp

添加 SSE server

axiomate mcp add --transport sse search https://example.com/sse

带环境变量或请求头

axiomate mcp add -e API_KEY=xxx my-server -- node server.js
axiomate mcp add --transport http api https://example.com/mcp -H "Authorization: Bearer xxx"

注意：如果你把一个 URL 直接传给 axiomate mcp add name https://...，但没有写 --transport http 或 --transport sse，Axiomate 会按 stdio server 处理。这通常不是你想要的。

配置格式

MCP 配置的核心结构是一个 mcpServers 对象。每个 key 是 server 名称，value 是连接配置。

stdio

{
  "mcpServers": {
    "filesystem": {
      "type": "stdio",
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-filesystem", "."],
      "env": {
        "API_KEY": "${MCP_API_KEY}"
      }
    }
  }
}

HTTP

{
  "mcpServers": {
    "remote-api": {
      "type": "http",
      "url": "https://example.com/mcp",
      "headers": {
        "Authorization": "Bearer ${MCP_TOKEN}"
      }
    }
  }
}

SSE

{
  "mcpServers": {
    "events": {
      "type": "sse",
      "url": "https://example.com/sse",
      "headers": {
        "X-Api-Key": "${MCP_API_KEY:-dev-key}"
      }
    }
  }
}
字段 适用 transport 说明
type 全部 stdio 可省略；远端 server 使用 http 或 sse。JSON 层也支持 ws，但普通 mcp add 目前不暴露 WebSocket 添加入口。
command / args stdio Axiomate 要启动的本地命令和参数。
env stdio 传给子进程的环境变量。值支持 ${VAR} 和 ${VAR:-default} 展开。
url http / sse / ws 远端 MCP endpoint。
headers http / sse / ws 静态请求头。值支持环境变量展开。
headersHelper http / sse / ws 运行一个本地 helper 动态生成 headers，适合短期 token 或公司内部凭证。
oauth http / sse 配置 MCP OAuth 相关参数，例如 clientId、callbackPort、authServerMetadataUrl。

Windows 上使用 npx：如果 project 配置里直接写 "command": "npx"，Axiomate 会给出 warning。更稳妥的写法是使用 cmd /c 包装：

{
  "type": "stdio",
  "command": "cmd",
  "args": ["/c", "npx", "-y", "my-mcp-server"]
}

配置范围

axiomate mcp add 默认写入 local scope。你可以用 -s 明确指定写到哪里。

scope 命令示例 用途
local axiomate mcp add -s local ... 当前项目下仅你自己使用。适合私有 token、本机路径、临时 server。
user axiomate mcp add -s user ... 写入用户全局配置，所有项目可用。
project axiomate mcp add -s project ... 写入当前目录的 .mcp.json，适合团队共享，但需要审批后才会连接。

如果同名 server 出现在多个 scope，实际聚合时后面的配置会覆盖前面的配置。常规优先级是：插件 < 用户 < 已批准的项目配置 < 本地配置。企业托管 MCP 配置存在时会独占控制，不再合并普通用户配置。

运行与管理

添加配置后，重启 Axiomate 或开启新会话，Axiomate 会自动加载 MCP 配置并连接可用 server。连接成功后，server 暴露的 tools 会加入模型可用工具池。

对话内管理

命令 作用
/mcp 打开 MCP 管理界面，查看 server 状态、工具数量、prompt、resource、认证状态。
/mcp reconnect <server-name> 手动重连指定 server。
/mcp disable <server-name> 禁用指定 server。禁用状态写入当前项目配置。
/mcp enable <server-name> 重新启用指定 server 并尝试连接。
/mcp disable / /mcp enable 不带 server 名时，对所有当前显示的 MCP server 生效。

命令行管理

命令 作用
axiomate mcp list 列出已配置 server，并做连接健康检查。
axiomate mcp get <name> 查看某个 server 的配置范围、类型、URL 或命令。
axiomate mcp remove <name> 删除 server 配置；远端 OAuth token 也会清理。
axiomate mcp reset-project-choices 清空当前项目对 .mcp.json server 的批准/拒绝记录。

健康检查会真的连接 server。axiomate mcp list 和 axiomate mcp get 为了检查状态，会跳过 workspace trust 对话并可能启动 .mcp.json 里的 stdio server。只在你信任的目录里运行这些命令。

认证与请求头

静态 headers

HTTP、SSE、WebSocket 配置可以直接写 headers。推荐用环境变量保存敏感值。

{
  "mcpServers": {
    "internal": {
      "type": "http",
      "url": "https://internal.example.com/mcp",
      "headers": {
        "Authorization": "Bearer ${INTERNAL_MCP_TOKEN}"
      }
    }
  }
}

动态 headersHelper

headersHelper 会在连接时运行一个本地命令。命令必须向 stdout 输出 JSON object，所有 value 都必须是字符串。

{
  "mcpServers": {
    "internal": {
      "type": "http",
      "url": "https://internal.example.com/mcp",
      "headersHelper": "node scripts/mcp-headers.js"
    }
  }
}
{
  "Authorization": "Bearer eyJ...",
  "X-Workspace": "payments"
}

helper 运行时会收到 AXIOMATE_CODE_MCP_SERVER_NAME 和 AXIOMATE_CODE_MCP_SERVER_URL 环境变量，方便一个 helper 服务多个 server。

OAuth

HTTP/SSE server 如果返回需要 OAuth 的响应，Axiomate 会把 server 标成 needs-auth，并在 /mcp 里提供认证入口。也可以在添加 server 时指定 OAuth 参数：

axiomate mcp add --transport http work https://example.com/mcp \
  --client-id my-client-id \
  --client-secret \
  --callback-port 1455

--client-secret 会提示你输入 client secret；非 TTY 环境下可以用 MCP_CLIENT_SECRET 环境变量提供。

项目共享 .mcp.json

团队可以把项目需要的 MCP server 写进仓库根目录的 .mcp.json。Axiomate 会从当前目录向上查找 .mcp.json，越靠近当前工作目录的配置优先级越高。

{
  "mcpServers": {
    "repo-tools": {
      "type": "stdio",
      "command": "cmd",
      "args": ["/c", "npx", "-y", "@example/repo-mcp"]
    }
  }
}

项目共享配置默认不会悄悄连接。第一次看到 project scope server 时，Axiomate 会要求你批准或拒绝。批准记录保存在你的本地项目配置里，不需要提交到仓库。

不要把密钥提交到 .mcp.json。团队共享配置里应该引用环境变量，例如 ${MCP_TOKEN}，让每个用户在本机设置自己的值。

高级入口

add-json

直接把一个 server JSON 写入配置，适合当前 CLI add 不覆盖的字段，例如 headersHelper 或 ws。

--mcp-config

启动 Axiomate 时从 JSON 字符串或 JSON 文件临时加载 MCP server，scope 为 dynamic。

插件 MCP

插件可以声明 mcpServers 或插件目录里的 .mcp.json。Axiomate 会给插件 server 加上 plugin:<plugin>:<server> 前缀。

add-json 示例

axiomate mcp add-json internal '{"type":"http","url":"https://example.com/mcp","headersHelper":"node scripts/headers.js"}'

--mcp-config 示例

axiomate --mcp-config ./mcp.local.json
axiomate --mcp-config '{"mcpServers":{"api":{"type":"http","url":"https://example.com/mcp"}}}'

--strict-mcp-config 会只使用 --mcp-config 传入的 server，忽略用户、项目、本地和插件配置。--bare 也会跳过自动发现的 MCP 配置，只保留显式传入的 --mcp-config。

安全模型

MCP server 可能执行本地代码、访问网络或读取系统资源。Axiomate 对 MCP 做了几层约束：

  • 项目 .mcp.json server 需要用户批准后才会连接。
  • 所有 MCP tool call 仍然经过 Axiomate 的工具权限与审批流程。
  • 当前项目可以禁用某个 server，禁用后不连接、不暴露工具。
  • 企业策略可以通过 allowedMcpServers / deniedMcpServers 限制 server 名称、命令或 URL。
  • 企业托管 MCP 配置存在时，可以独占控制全部 MCP server。

实务建议：只添加你信任的 server；不要运行来源不明的 stdio command；不要把 token 写进仓库；对团队共享配置优先使用环境变量和最小权限 token。

排错

现象 检查点
/mcp 显示没有 server 先运行 axiomate mcp list；确认配置写入了正确 scope；如果是 .mcp.json，确认 server 已批准。
URL server 一直失败 确认添加时写了 --transport http 或 --transport sse；检查代理、证书、请求头和服务端 endpoint。
显示 needs-auth 进入 /mcp 选择该 server 完成认证；如果 token 失效，重新认证或删除后重加。
环境变量 missing warning 配置里用了 ${VAR}，但当前 shell 没有设置这个变量。设置变量后重启 Axiomate，或使用 ${VAR:-default}。
Windows 上 npx server 启动失败 把 command 改成 cmd，把 args 改成 ["/c", "npx", ...]。
改了配置但工具没变化 在 /mcp 里 reconnect，或重启 Axiomate。插件 MCP 变化可尝试重新加载插件。

调试命令：用 axiomate --debug 启动后再打开 /mcp，失败 server 的连接错误会更容易定位。
