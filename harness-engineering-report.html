Manus 原则收官 —— 为什么 PR 2-5 全部不做

Manus 2025-11 "Context Engineering" 文章的核心论点：

AI 犯错的历史记录本身就是 learning signal。压缩掉"失败的尝试"等于在抹去"为什么不那么做"的教训。下一轮 LLM 失去了自我纠错的凭据，会重复同样错误。

这个原则直接否定了 PR 2-5 里所有"通过裁剪历史节省 token"的优化：

PR原方案v4 决定理由
PR 1 Iterative prompt + prev summary 注入 ✅ 已落地 不抹历史，只是更稳的 update vs rewrite。Manus 原则中性。
PR 2 Pre-pass tool-aware 结果剪枝 ❌ 不做 裁掉 tool output 等于裁掉 LLM 自我纠错依据。"npm test → 47 lines"摘要丢的是错误堆栈/重现步骤。
PR 3 compact 走 midModel + 静态 fallback ❌ 不做 #6：主模型做 compact 是合理的（quality > cost）。#8：实地挖代码后发现 axiomate 现有 fail-fast (notification + 3 次熔断 + reactive compact) 比 hermes 静默静态消息健康得多 —— hermes 的"占位 + 删消息"会让模型在无摘要状态下继续瞎聊，净值为负。
PR 4 dedup + truncate + anti-thrash + redaction ❌ 不做 dedup 二元组会丢历史快照（"原先是 user.token"信息没了）；三元组本身命中率 <10%（Read 已被 FILE_UNCHANGED_STUB tool-level 处理）。truncate 巨型 paste 里的 stack trace / 重现步骤是高价值信号。anti-thrash 与现有 3 次熔断重叠。
PR 5 /compact <topic> budget 分配 ❌ 暂不做 focus-topic 会更激进按"相关性"砍内容，更容易丢学习信号。如果未来要做，思路应是"扩相关段 budget"而非"削无关段"。

替代路径：需要省 token 时走 1) 扩 context window（换更大模型）；2) 提早触发 compact（让每次 compact 做得更"浅"）；3) tool-level idempotent stub（如 Read 的 FILE_UNCHANGED_STUB，在 tool 执行阶段消重，不是 compact 后处理）。

影响：compact feature parity 在 v4 从 v3.2 的 ~40% 推到 ~50%（PR 1 关掉 #1 #2），剩余 8 个 gap 出于哲学原因永久放弃。这不是"懒得做"，是"做了反而是负价值"。

🔍 v4.1 后续概念纠错（2026-04-24 同日）

用户深入提问揭示三个早期分类错误，本节集中纠正：

  1. Auxiliary Client ≠ Smart Routing：原 v3 报告隐性把两者归并为"模型层架构差距"。实测后是两个独立机制：
    • Auxiliary Client = side-task router（compression / search / vision 用哪个 backend）—— hermes 仍重投，2,771 行 active
    • Smart Routing = user-turn classifier（用户 message 走 cheap 还是主 model）—— hermes 已删除（424e9f36，2026-04-19）
    见上方 GAP 3 / GAP 4 改写。
  2. Hermes "Code Execution Sandbox" ≠ axiomate sandbox-axiomate：前者是 PTC（Programmatic Tool Calling）的 child process 进程隔离，后者是 OS-level 安全沙箱。两个项目对 Windows 都是 disable —— 不是 axiomate gap，是行业现实。Windows 缺 unprivileged sandbox primitives：
    • Windows Sandbox / Hyper-V Container：分钟级启动 + 要 Pro/Enterprise + 虚拟化
    • Job Objects：能逃逸，不算真隔离
    • WSL2+seccomp：能用但加 WSL 强依赖
    • AppContainer：实现复杂度极高（manifest / capability registration / Windows-only API）
    • Docker Desktop：重依赖 + 性能损耗
    Hermes 和 axiomate 同选 disable + 提示用户。这条不计 gap。
  3. Smart Routing 的状态从"v3 推荐 port" 变成"hermes 自己删了" —— 强烈负信号。axiomate 不要 port。替代方案：/fast slash command（用户主动）。
