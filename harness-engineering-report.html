# Harness 模式 Hermes Axiomate 差异说明
6 Credential Pooling Advanced 缺失 GAP Hermes credential_pool.py (1,326 行): 多 credential per provider + 4 rotation strategy + OAuth refresh。Axiomate: 单 key per model
14 Stale Connection Detection Intermediate Intermediate 两者功能相当。Hermes: last_chunk_time + dynamic stale timeout (300-600s)。Axiomate stallDetection.ts (71 行) + ECONNRESET/EPIPE 检测
15 Provider Abstraction Advanced Intermediate Hermes: 3 API modes (chat_completions/anthropic_messages/codex_responses) + 6-level auxiliary fallback chain。Axiomate provider.ts: neutral StreamRequest/Response，anthropicProvider.ts (675 行) + openaiProvider.ts (425 行)，无 fallback 链
16 Subagent / Delegation Advanced Partial Hermes delegate_tool.py: credential inheritance + toolset restriction + heartbeat (30s) + depth limit=2。Axiomate AgentTool/runAgent.ts + forkedAgent.ts: 状态隔离但缺 depth limit / heartbeat / credential inheritance
17 Cross-Session Memory Advanced Partial Hermes memory_manager.py (373 行): pluggable providers + lifecycle hooks。Axiomate sessionMemory.ts (450 行): file-based auto-memory，无 provider 架构
18 Auxiliary LLM Client Advanced 部分实现 Hermes auxiliary_client.py (3,144 行): 6 级 fallback + per-task override。Axiomate: 三级模型 (currentModel / midModel / fastModel) + 自动 fallback 链 + 内部任务路由 cheap tier — 重叠 >70%。

真适用 axiomate 的 10 项 gap：
1. 3 级 vs 6 级 fallback chain（per-task config → primary aux → secondary aux → main → cheap → local）
2. 无 per-task 配置表（哪个任务用哪个 tier 硬编码在调用点，不能 config 改）
3. 无 credit exhaustion tracking（hermes 检测到额度耗尽自动切下一 provider；axiomate errorClassifier 能分辨 billing/auth 但未联动 fallback）
4. 无 vision provider 单独 resolution（hermes resolve_vision_provider_client() + _PROVIDER_VISION_MODELS 映射 xiaomi→mimo-v2.5 / zai→glm-5v-turbo 等；axiomate sideQuery 完全 provider-agnostic）
5. 无 per-task timeout（auxiliary.{task}.timeout 让 vision 120s / compression 60s 分开配；axiomate 固定 default）
6. 无 per-task extra_body（auxiliary.{task}.extra_body 加 reasoning_effort 等；axiomate 无 per-task HTTP body 定制）
7. 无跨 provider 图像格式转换（hermes 自动 OAI image_url ↔ Anthropic image，给 OAI 协议端点发 Anthropic 图会转；axiomate 不转，直接挂）
8. 无 max_tokens 参数名预协商（hermes 请求前按 base_url 选 max_tokens 还是 max_completion_tokens；axiomate withRetry.ts:251-275 仅事后 400 错回退，多一轮失败）
9. 无模型特定参数约束（hermes 检测 Kimi server-managed temperature → strip / Opus 4.7+ 拒采样参数 → strip；axiomate 一律照发，遇特殊模型可能 400）
10. （上面 6+4 = 10 项真 gap 完整版）

不适用 axiomate 的 4 项（hermes 有但与 axiomate 场景无关）：
- Nous Portal 401 token refresh（axiomate 不接 Nous Portal）
- Codex Responses API adapter（axiomate 仅支持 Anthropic + OpenAI 协议）
- Client cache async 事件循环绑定（Python asyncio 特有，Node 无等价问题）
- Nested AsyncOpenAI cleanup（Python GC + httpx 特有）

核心架构差异：hermes 把 aux 当一等子系统（每 task 类型可单独配 timeout / extra_body / 模型 / fallback 链）；axiomate 走统一 sideQuery() 路径，假设所有 aux task 等价于主请求路径。这个假设在 vision / compression 这种"可以等更慢更便宜"的任务上是错的
19 Skill & Memory Auto-Distillation Advanced 部分实现 Hermes 3 层架构：SKILLS_GUIDANCE 系统提示 + skill_manager_tool.py (817 行) 6 action 暴露给 LLM 自主写 + 每 N 轮 (默认 10) 后台 fork-agent 跑 review prompt 自动决策存/更/不存。Axiomate 有 /skillify bundled skill（用户主动）+ skill loader + chokidar watcher + forkedAgent 基础设施全在；缺 LLM 自主版 skill_manage tool + 周期 fork-review 触发器
